# 社内タスク・案件管理システム — 開発チケット一覧

## 概要

本ドキュメントは、FORMAWORK.ai（美容室向けAI CRM）を社内タスク・案件管理システムへ作り替えるための開発チケット一覧です。

**既存コードベースを最大限流用** しつつ、不要機能の削除・新機能の追加を行う方針とします。

---

## コードベース現状サマリー

| 既存リソース | 転用方針 |
|---|---|
| Supabase Auth（ログイン/ログアウト） | **そのまま流用** |
| スタッフ管理機能 | **ユーザー管理として流用・拡張** |
| 管理者ロール（admin/user） | **そのまま流用** |
| shadcn/ui コンポーネント群 | **そのまま流用** |
| Drizzle ORM + PostgreSQL | **スキーマ変更して流用** |
| Valibot + react-hook-form | **そのまま流用** |
| 顧客管理 CRUD 構造 | **案件管理の実装参考として流用** |
| customer_notes の 1:N 構造 | **タスク管理の実装参考として流用** |
| AI 機能（メモリー生成・アドバイス） | **削除** |
| ランディングページ | **削除** |
| Cron ジョブ（AI 処理） | **削除** |
| 顧客テーブル群 | **削除（新スキーマに置換）** |

## 実装方針メモ

- ルートは `/customers` → `/projects`、`/staffs` → `/users` に変更する
- DB スキーマは既存テーブルを削除し、新スキーマで置き換える（マイグレーション）
- AI 機能（Cron・API Routes）は最初に削除してから他の作業に着手する
- スタッフ管理の CRUD・ページ構造は、ユーザー管理・案件管理の実装テンプレートとして活用する
- Supabase Auth の仕組み（auth_metadata の role/staffId）は流用し、staffId を userId の別名として扱う
