# 基本方針

## 基本方針

シークレットにエージェントがアクセスできない状態を作り、情報漏洩を防止する。
シークレット管理だけでは完全にデータの流出を防げないため、アクセスできるネットワークを制限し、多段的な防御を行う前提とする。

### ローカル開発では本番環境のシークレットに触れさせない

ローカル開発環境やモックを構築し、本番環境のシークレットを使わなくても開発できるようにする。
原則としてコーディングエージェントを動かす環境では、本番環境に接続しない構成を優先する。

### .env に機密情報を含めない

deny 設定で Read(.env) を指定すればエージェントのアクセスを防げるが、.env を読み込む Bash コマンドも実行できなくなり開発効率が下がるため、この設定は行わない。
代わりに、そもそも .env に機密情報を含めないことで対処する。
OWASP Secrets Management Cheat Sheet でもローカルにシークレットを平文保管しないことが推奨されている。

### 環境変数にシークレットを直接含めない

シークレットは Secret Manager（AWS Secrets Manager、GCP Secret Manager、HashiCorp Vault など）を使い、ランタイムで注入する。
OWASP Secrets Management Cheat Sheet でも環境変数にシークレットを直接含めないことが推奨されている。

ただし、ランタイムで注入しても、エージェントが console.log 等を挿入して内容を取得できる可能性があり、完全には防げない。
