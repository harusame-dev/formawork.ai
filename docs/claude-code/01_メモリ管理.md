# メモリ管理

## CLAUDE.md

### 参考

- https://github.com/abhishekray07/claude-md-templates/blob/main/principles.md
- https://www.humanlayer.dev/blog/writing-a-good-claude-md
- https://code.claude.com/docs/en/best-practices

### 基本方針

- なるべく小さく保つ。最大でも 300 行未満に保つ
- エージェントが調べればわかる内容は記載しない
- `@file` は原則使用しない（分割していてもセッション開始時に読み込まれコンテキストが増えるため）
- 特定のフォルダ配下でのみ有効な内容はモジュール用の CLAUDE.md を作成
  - 例：`apps/web/CLAUDE.md`
- 特定のファイルパスパターンでのみ有効なルールは `.claude/rules` 配下に配置
- 特定の状況で必要な情報は `agent_docs` 配下に配置して必要な時に読み込ませる

### 記載内容

#### リポジトリ概要

リポジトリの簡単な説明

#### フォルダ構成

大まかな構成のみ（詳細はエージェントが調べる）  
モノレポの場合はパッケージ名も記載

#### コマンド

エージェントが開発で使うプロジェクト固有のコマンドのみ

#### プロジェクト固有の情報

- 開発サーバー情報
- ブランチ命名規則

#### AI エージェントのワークフロー

- タスク実行時にやること
- 完了基準

#### 参照ドキュメント

- `agent_docs` 配下のフォルダを参照すること、必要に応じて読み込むことのみ記載
- 実際のファイル一覧は hooks によりセッション開始時に注入（ファイル増減のたびの手動更新を防ぐため）

## .claude/rules

### 参考

- https://code.claude.com/docs/ja/memory#claude/rules/-でルールを整理する

### 基本方針

特定のファイルパスパターンにのみ適用したいルールを記載する。  
プロジェクト全体に適用するルールは CLAUDE.md に記載する。

### ファイル一覧

フレームワークで規定されていない独自ルール

| ファイル            | 適用パターン                            |
| ------------------- | --------------------------------------- |
| server-action.md    | `*.sa.ts`                               |
| server-component.md | `*.sc.tsx`                              |
| client-component.md | `*.cc.tsx`                              |
| browser-test.md     | `*.browser.test.ts`, `*.browser.ui.tsx` |
| server-test.md      | `*.server.test.ts`                      |
| e2e-test.md         | `e2e/*.e2e.ts`                          |

## agent-docs

https://vercel.com/blog/agents-md-outperforms-skills-in-our-agent-evals

### 基本方針

- CLAUDE.md には `agent-docs` を必要に応じて読み込むよう記載し、エージェントが自律的に判断して読み込む
- ファイル名は日本語で、名前だけで内容がわかるようにする
- コンテキストが肥大しないよう、関連する情報のみをコンパクトに記載する
- agent skill でもリファレンススキルとして同じような使いができるが、呼び出し精度が現状低いため、 agent-docs を優先する
