# AI 駆動開発

セキュリティー・品質を担保した状態で開発スピード向上・エンジニアの工数削減を目指す

## 基本方針

以下を達成するために各設定を行う

- エンジニアの介入は最小限で開発できる
  - エンジニアに実行許可を求めない
  - コンテキストを手動で渡す必要がない
    - チケット、
    - CI / CD 結果
    - 設計方針
    - ブラウザの情報
    - 開発サーバーの情報
    - 開発 DB の情報
    - オブザーバビリティの情報
  - 実装コードが正しいかハーネスにより検証可能
    - Lint
    - Format
    - デッドコードチェック
    - テスト
    - テストカバレッジ
    - Light House スコア
    - セキュリティチェック
- セキュリティーが担保されている
  - 破壊的なコマンドを実行できない / 実行できても問題ない
  - 重要な情報が外部に流出しない
- 品質が担保されている
  - 設計方針が守られている
  - 不具合のないコードが生成される

## 開発フロー

### 通常開発フロー

| 担当 | 作業内容                   | 備考                                                                                    |
| ---- | -------------------------- | --------------------------------------------------------------------------------------- |
| 人   | ブランチを切る             | チケット番号をブランチ名に含める<br>例: `git switch -c 156-foo-feature-implementation`  |
| 人   | 実装計画を立てる指示を出す | `/imp-plan`                                                                             |
| AI   | 実装計画を立てる           | チケットの内容・ユーザーからの指示をもとに実装計画を立てる                              |
| 人   | 実装計画をレビューする     | 問題がなくなるまで繰り返す                                                              |
| 人   | 実装を指示する             | 変更量が多いときはステップ実行も検討。その場合、事前にチケット分割も考慮                |
| AI   | 実装する                   | 基本的にエンジニアに確認を求めない。ハーネス（lint など）が通るまで実装を自律的に進める |
| AI   | コミットする               |                                                                                         |
| 人   | コードレビューと修正       | 問題点があればAIもしくは手動で修正可                                                    |
| 人   | PR 作成を依頼する          | `/pr`                                                                                   |
| AI   | PRを作成する               | チケットの内容、実装内容差分をもとに PR を作成                                          |

- PR 作成以降のフローへ続く

### 軽微な開発フロー

| 担当 | 作業内容                             | 備考                                                                                         |
| ---- | ------------------------------------ | -------------------------------------------------------------------------------------------- |
| 人   | チケット上で実装を指示する           |                                                                                              |
| AI   | 不明点があればコメントする           | チケット上でコメントする                                                                     |
| 人   | コメントに返信し、再度実装を指示する | 実装開始できるまで繰り返す。<br>複数回往復が必要な複雑なタスクは任せず、通常の開発に移行する |
| AI   | ブランチを切って実装する             | チケット番号をブランチ名に含める。ハーネスが通るまで自律的に実装をする                       |
| AI   | PR を作成する                        |                                                                                              |

- PR 作成以降のフローへ続く

### PR 作成以降のフロー

| 担当       | 作業内容                                  | 備考                                                              |
| ---------- | ----------------------------------------- | ----------------------------------------------------------------- |
| AI         | 1 次レビュー                              | PR にコメント。PR 後に担当者に通知                                |
| AI         | CI 失敗した場合は原因を調査・コメントする |                                                                   |
| 実装者     | 2 次レビューをする                        |                                                                   |
| 実装者     | 適用する指摘内容を指定し、修正を指示する  | PR 上で行う。修正が多い場合や手数がかかる場合はローカルで開発する |
| AI         | 実装して push する                        |                                                                   |
| AI・実装者 | レビューと修正を繰り返す                  | 問題がなくなるまで継続する                                        |
| 実装者     | メンバーにレビュー依頼を行う              |                                                                   |
| メンバー   | レビューを行う                            |                                                                   |
| AI・実装者 | 修正があれば対応し、再度レビュー依頼する  | 問題がなくなるまで継続する                                        |
| メンバー   | Approve する                              |                                                                   |
| 実装者     | マージする                                |                                                                   |
