# 外部サービス

## 基本方針

### コーディングエージェントに渡す権限を絞り、権限レベルで操作を制限する

eg: GitHub で Administrator 権限のない PAT を発行して gh を使う

### 外部サービス側で破壊的な変更や問題のある操作できないように設定する

- eg: GitHub で保護ブランチに直接 push できないようにする

### deny list で問題のあるコマンドを実行できないようにする

- 迂回する可能性があるため補助的に利用。
- eg: gh repo delete

### 何処かの設定が漏れたり迂回する可能性を考慮して多段防衛的に設定

- Github でリポジトリを削除できない権限の PAT を利用した上で deny list に登録する

## Github

### ツール

- gh

### 権限

以下の権限を許可した fine-grained PAT を発行し、 GITHUB_TOKEN 環境変数に設定。
普段は OAUTH した権限 gh を使えるように ~/.claude/settings.json の env フィールドに登録を推奨。

- Actions
- Contents（read のみ）
- Issues
- Metadata（read のみ）
- Pull requests

### 設定

- 保護ブランチへの push 禁止
- 保護ブランチへの強制 push 禁止
- マージ時は 1 人以上の認証を必須

### deny list

curl などで直接 api.github.com を叩かれてしまうと実行できてしまうので、あくまで補助的に使用
