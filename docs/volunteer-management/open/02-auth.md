# チケット 02: 認証（ログイン・ログアウト）流用・調整

## 概要

既存のログイン・ログアウト機能を流用し、ボランティア来場管理システム向けに調整する。
ログイン後のリダイレクト先をイベント一覧（`/events`）に変更する。

## 背景

Supabase Auth を使用した認証機能はすでに実装済みであり、再実装は不要。
ただし、ログイン後のリダイレクト先が旧システム向けになっているため、調整が必要。
また、ナビゲーション・レイアウトを新システム向けに刷新する。

## 実装対象

### 調整箇所

| 対象 | 変更内容 |
|---|---|
| ログイン後リダイレクト先 | 旧トップページ → `/events` |
| 認証エラー時リダイレクト | `/login` のまま維持 |
| `(private)` レイアウト | ナビゲーションをイベント一覧のみにシンプル化 |
| ログアウト後リダイレクト | `/login` のまま維持 |

### ナビゲーション設計

認証済みユーザー向けのサイドバー・ヘッダーを以下のシンプルな構成に変更：

```
[ロゴ / アプリ名]
─────────────────
イベント一覧    ← /events へのリンク
─────────────────
ログアウト
```

### 保持する機能

- Supabase Auth セッション管理
- middleware による未認証リダイレクト
- ログインフォーム（メール・パスワード）
- ログアウト処理

## 実装手順

1. `apps/web/app/(private)/layout.tsx` のナビゲーションをイベント一覧のみにシンプル化
2. ログイン成功後のリダイレクト先を `/events` に変更
3. `pnpm -w validate:check` でエラーがないことを確認
4. `pnpm -w build` でビルドが通ることを確認
5. ブラウザで動作確認
   - ログイン → `/events` にリダイレクトされること
   - 未認証で `/events` にアクセス → `/login` にリダイレクトされること
   - ログアウト → `/login` にリダイレクトされること

## 完了条件

- [ ] ログイン後に `/events` にリダイレクトされる
- [ ] 未認証でプライベートページにアクセスすると `/login` にリダイレクトされる
- [ ] ログアウトが正常に動作する
- [ ] ナビゲーションがボランティア管理システム向けの構成になっている
- [ ] `pnpm -w validate:check` が通る
- [ ] `pnpm -w build` が通る
- [ ] ブラウザでの動作確認が完了している（スクリーンショットで画面崩れがないこと）
