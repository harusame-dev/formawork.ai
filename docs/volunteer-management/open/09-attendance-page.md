# チケット 09: 来場画面（ボランティア向け、認証不要）

## 概要

ボランティアが来場記録を行う画面を実装する。
イベント固有のトークン付きURLでアクセスでき、認証は不要。
据え置き端末での利用を想定した UI とする。

## 背景

会場の受付等にタブレットを設置し、ボランティアが順番にIDを入力して来場記録を行う運用を想定。
管理者のセッションに依存しない独立したページとして実装する。

## 実装対象

### 画面

| URL | 内容 |
|---|---|
| `/attendance/[token]` | 来場画面（認証不要） |

### 画面フロー

```
[1. ID入力画面]
  - 6桁数字コードの入力フォーム
  - 「確認する」ボタン

    ↓ コード照合（DB検索）

[2a. 本人確認画面]（照合成功）
  - 「○○さん、こんにちは」と氏名表示
  - 「来場を記録する」ボタン

    ↓ ボタン押下

[2b. エラー画面]（照合失敗）
  - 「IDが見つかりませんでした」
  - 「再入力する」ボタン → [1] に戻る

[3. 完了画面]
  - 「来場を記録しました」と表示
  - 記録日時を表示
  - 「次の人へ」ボタン → [1] に戻る（IDフィールドをクリア）
```

### セキュリティ仕様

- トークンが存在しない場合: 404 ページを表示
- ボランティアの照合はイベントID（トークン経由で取得）と code の組み合わせで行う
- 来場記録は回数制限なし（1日複数回押せる）

### UI 設計方針

- 据え置き端末（タブレット横向き等）での利用を想定
- 大きな文字・ボタンで操作しやすい UI
- 数字入力は大きな数字キーボード（`inputMode="numeric"` 等）

## features モジュール構成

```
apps/web/features/attendance-page/
  components/
    code-input-form.tsx        # ID入力フォーム
    volunteer-confirm.tsx      # 本人確認・来場ボタン
    attendance-complete.tsx    # 完了画面
  actions/
    lookup-volunteer.ts        # コード照合
    record-attendance.ts       # 来場記録
  queries/
    get-event-by-token.ts      # トークンからイベント情報取得
```

## ルーティング

`/attendance/[token]` は `(public)` グループに配置し、認証ミドルウェアの対象外とする。

```
apps/web/app/(public)/attendance/[token]/
  page.tsx
```

## 実装手順

1. `features/attendance-page/` モジュールを作成
2. `get-event-by-token.ts` を実装
   - token から `event_attendance_urls` → `events` を取得
   - token が存在しない場合は null を返す（ページで 404 処理）
3. `lookup-volunteer.ts` サーバーアクションを実装
   - eventId + code でボランティアを検索
4. `record-attendance.ts` サーバーアクションを実装
   - `attendance_records` に INSERT
5. 各コンポーネントを実装
6. `/attendance/[token]` ページを実装
   - token が存在しない場合は notFound() を呼び出す
7. ミドルウェアで `/attendance/` パスを認証対象外に設定
8. `pnpm -w validate:check` でエラーがないことを確認
9. `pnpm -w build` が通ることを確認
10. ブラウザで動作確認
    - 有効なトークンでアクセス → 来場画面が表示される
    - 無効なトークンでアクセス → 404 が表示される
    - 正しいコード入力 → 本人確認画面が表示される
    - 誤ったコード入力 → エラーが表示される
    - 「来場を記録する」ボタン → 完了画面が表示される
    - 「次の人へ」ボタン → ID入力画面に戻る
