# 命名規則

ファイルパスからそのファイルの種別を識別できるよう、以下の命名規則を遵守する。

## ファイル種別と命名パターン

| ファイル種別               | 命名パターン              | 説明                                                                                    |
| -------------------------- | ------------------------- | --------------------------------------------------------------------------------------- |
| サーバーアクション         | `*.action.ts`             | Next.js Server Actions（`"use server"` 宣言）                                           |
| サーバーコンポーネント     | `*.server.tsx`            | サーバーサイドのみで実行される `async` コンポーネント                                   |
| クライアントコンポーネント | `*.client.tsx`            | `"use client"` 宣言があるコンポーネント                                                 |
| ユニバーサルコンポーネント | `*.universal.tsx`         | `"use client"` なし・`async` なし。サーバー/クライアント両方で動作するコンポーネント   |
| React hook                 | `use-*.hook.ts`           | `use` プレフィクスの React カスタムフック                                               |
| ブラウザテスト             | `*.browser.test.{ts,tsx}` | Vitest Browser Mode テスト（サイズプレフィクス推奨: `*.{small,medium}.browser.test.*`） |
| サーバーテスト             | `*.server.test.ts`        | Vitest サーバーサイドテスト（サイズプレフィクス推奨: `*.{small,medium}.server.test.ts`）|
| E2E テスト                 | `*.e2e.test.ts`           | Playwright E2E テスト（サイズプレフィクス推奨: `*.{medium,large}.e2e.test.ts`）         |

## 各種別の詳細

### サーバーアクション（`*.action.ts`）

```typescript
"use server";
export async function createUser(data: FormData) { ... }
```

- `"use server"` ディレクティブを含む
- フォームアクションや Server Actions として使用される関数を定義

### サーバーコンポーネント（`*.server.tsx`）

```typescript
export async function UserListServer() {
  const users = await fetchUsers();
  return <UserList users={users} />;
}
```

- `async function` で定義されたコンポーネント
- データフェッチを行うコンテナ相当のコンポーネント
- `"use client"` を含まない

### クライアントコンポーネント（`*.client.tsx`）

```typescript
"use client";
export function SearchForm() {
  const [query, setQuery] = useState("");
  ...
}
```

- `"use client"` ディレクティブを含む
- インタラクティブな UI コンポーネント

### ユニバーサルコンポーネント（`*.universal.tsx`）

```typescript
export function UserCard({ user }: { user: User }) {
  return <div>{user.name}</div>;
}
```

- `"use client"` なし、`async` なし
- サーバー/クライアント両方でレンダリング可能
- スケルトン（`*-skeleton.universal.tsx`）や純粋な表示コンポーネントが該当

### React hook（`use-*.hook.ts`）

```typescript
export function useImageUpload() {
  const [file, setFile] = useState<File | null>(null);
  ...
}
```

- ファイル名・関数名ともに `use-` / `use` プレフィクスあり
