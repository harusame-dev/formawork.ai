# 外部ライブラリの使用

## ラッパーの作成

外部ライブラリは API の変更や、ライブラリの切り替えなどで変更が発生する可能性がある。
そのため原則として外部ライブラリはアプリに必要なユースケースに絞った薄いラッパーを作成し、直接呼び出さない。
ただし、過度に複雑になってしまう場合は直接呼び出しも許容する。

### 外部ライブラリを直接使用する例

直接呼び出しているため、もし date-fns のアップデートが止まって別のライブラリに乗り換えたり、api の変更でフォーマット文字列が変わったときなど、使用箇所全ての更新が必要になる。

```ts
import { format } from "date-fns";

function main() {
  // date-fns の format を使って日付を出力している
  const date = format(new Date(), "yyyy-MM-dd");
}
```

### 外部ライブラリのラッパーを作成する例

ラッパーを挟んでいるため API を切り替えるときなども date.ts の実装を修正すれば良い。
また、ラッパーのテストを書くことで依存ライブラリの自動更新でも破壊的変更がないことをチェックできる。

```ts
// date.ts
import { format } from "date-fns";

function formatDate(date: Date): string {
  return format(date, "yyyy-MM-dd");
}

// main.ts
import { formatDate } from "./date";

function main() {
  const date = formatDate(new Date());
}
```
