# テスト： 共通

## テストでは実装のからのインポート避け、テスト内で定義する

悪い例

```ts
import { MAX_ITEM_COUNT } from "./constants";

expect(items).toHaveLength(MAX_ITEM_COUNT);`
```

良い例

```ts
// もしくはテスト内で MAX_ITEM_COUNT を定義
expect(items).toHaveLength(100);
```

**理由:**

1. **意図しない変更の検出**: 実装の定数が変わったときにテストが壊れることで、変更に気づける。インポートしていると自動追従してしまい、バグを見逃す
2. **テストは仕様のドキュメント**: 「100件表示される」は仕様であり、テストで明示すべき。`${MAX_ITEMS}件表示される` は実装詳細の追従であり仕様ではない

## 実装ではなく、ユーザーから見た振る舞い（外部から観察可能な動作）をテストする

- ❌ 悪い例: 「maxlength属性が設定されている」
- ✅ 良い例: 「最大文字数を超えて入力できない」

## describe の代わりに test.extend を使う

共通のセットアップ、ティアダウンを行いたい場合は test.extend を使ったフィクスチャーパターンを使用する
